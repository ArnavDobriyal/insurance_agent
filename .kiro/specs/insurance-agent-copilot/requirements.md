# Requirements Document

## Introduction

The Insurance Agent AI Copilot is a command-first, AI-driven workspace application designed for insurance agents in India. The system combines modern UI/UX principles (inspired by Linear, Notion AI, Arc Browser, and Apple VisionOS) with Google Gemini-powered AI capabilities to streamline field workflows, ensure IRDAI compliance, and maximize agent productivity through voice-first interactions, intelligent automation, and context-reactive interfaces.

## Glossary

- **System**: The Insurance Agent AI Copilot application
- **Agent**: The human insurance agent user of the System
- **AI Engine**: The Google Gemini-powered conversational and reasoning component
- **Compliance Guard**: The IRDAI rules engine middleware that validates all AI outputs
- **AutoPilot**: The autonomous workflow mode that executes queued tasks with user oversight
- **Command Bar**: The primary text/voice input interface for navigation and commands
- **Smart Context Panel**: The dynamic right-side panel that displays contextual information
- **Lead**: A potential or existing insurance customer record in the CRM
- **Lead Temperature**: A color-coded indicator of lead engagement level (hot, warm, cold)
- **Work Zone**: A categorized collection of leads or tasks (e.g., Hot Leads, Renewals)
- **IRDAI**: Insurance Regulatory and Development Authority of India
- **CRM**: Customer Relationship Management system
- **STT**: Speech-to-Text conversion service
- **Activity Feed**: The live log of AutoPilot actions and changes

## Requirements

### Requirement 1

**User Story:** As an insurance agent, I want a voice-first command interface, so that I can interact with the system hands-free while in the field

#### Acceptance Criteria

1. WHEN the Agent activates the voice assistant, THE System SHALL capture audio input and convert it to text using STT
2. WHEN the Agent speaks a command, THE System SHALL detect the intent and execute the corresponding action within 2 seconds
3. WHEN the Agent speaks in Hindi, Marathi, or Tamil, THE System SHALL detect the language and process the command in that language
4. THE System SHALL display a real-time transcript of the Agent's speech in an overlay panel
5. WHEN voice input is active, THE System SHALL display a pulsing glow animation on the voice assistant bubble

### Requirement 2

**User Story:** As an insurance agent, I want a centralized command bar, so that I can quickly navigate, search, and execute actions without multiple clicks

#### Acceptance Criteria

1. THE System SHALL display a command bar at the top of every screen with a rounded, glowing border on focus
2. WHEN the Agent types in the command bar, THE System SHALL provide autocomplete suggestions with AI-powered predictions
3. WHEN the Agent selects a command suggestion, THE System SHALL execute the action or navigate to the requested screen within 300 milliseconds
4. THE System SHALL support commands for navigation, lead creation, summaries, and CRM updates through the command bar
5. WHEN the command bar receives focus, THE System SHALL apply a soft glow animation with 200 millisecond duration

### Requirement 3

**User Story:** As an insurance agent, I want an AI-powered dashboard that shows my daily priorities, so that I can focus on high-value activities

#### Acceptance Criteria

1. WHEN the Agent opens the home dashboard, THE System SHALL display a Today's Briefing Card with a bullet-point summary of urgent items
2. THE System SHALL display horizontal swipeable Work Zone cards for Hot Leads, Renewals, Follow-ups Today, and Pending Tasks
3. WHEN the Agent taps a Work Zone card, THE System SHALL navigate to the filtered lead list for that category within 260 milliseconds
4. THE System SHALL display an AutoPilot toggle button on the home dashboard
5. THE System SHALL display a Quick Create Lead button on the home dashboard

### Requirement 4

**User Story:** As an insurance agent, I want to view detailed lead profiles with AI insights, so that I can understand customer context and take informed actions

#### Acceptance Criteria

1. WHEN the Agent selects a lead, THE System SHALL display the lead profile page with name, age, location, tags, and quick action buttons
2. THE System SHALL display a Last Interaction Summary generated by the AI Engine on the lead profile
3. THE System SHALL display a Sentiment Timeline Graph with 4 to 6 data points colored in Info Blue
4. THE System SHALL display a Conversion Probability Gauge as a semi-circular radial gauge
5. THE System SHALL display a chronological timeline of all interactions with the lead

### Requirement 5

**User Story:** As an insurance agent, I want AI-generated next best actions, so that I know exactly what to do for each lead

#### Acceptance Criteria

1. WHEN the Agent views a lead profile, THE System SHALL display a Next Best Action card in the Smart Context Panel
2. THE System SHALL generate the Next Best Action using the AI Engine with a confidence score above 70 percent
3. THE System SHALL display the action steps in a numbered list with color-coded difficulty indicators
4. THE System SHALL provide an explanation of why the action is recommended in the card
5. WHEN the Agent taps the Next Best Action, THE System SHALL prefill relevant templates or initiate the suggested workflow

### Requirement 6

**User Story:** As an insurance agent, I want predicted objections with safe responses, so that I can handle customer concerns confidently and compliantly

#### Acceptance Criteria

1. WHEN the Agent views a lead profile, THE System SHALL display an Objection Predictor panel in the Smart Context Panel
2. THE System SHALL list 3 to 5 predicted objections based on lead history and product type
3. WHEN the Agent taps a predicted objection, THE System SHALL display an IRDAI-compliant response script
4. THE System SHALL pass all response scripts through the Compliance Guard before display
5. THE System SHALL highlight the source signals used to predict each objection

### Requirement 7

**User Story:** As an insurance agent, I want pre-approved message templates, so that I can communicate quickly while staying compliant

#### Acceptance Criteria

1. WHEN the Agent views a lead profile, THE System SHALL display a Templates panel in the Smart Context Panel
2. THE System SHALL provide a search bar to filter templates by keyword or category
3. WHEN the Agent selects a template, THE System SHALL auto-fill dynamic values such as premium amount and due date
4. THE System SHALL display the data source for each auto-filled value in the preview
5. THE System SHALL pass all template content through the Compliance Guard before allowing the Agent to send

### Requirement 8

**User Story:** As an insurance agent, I want compliance warnings on risky content, so that I avoid IRDAI violations

#### Acceptance Criteria

1. WHEN the AI Engine generates content containing risky phrases, THE Compliance Guard SHALL detect the violation and flag it
2. WHEN a compliance violation is detected, THE System SHALL display a Compliance Flag Badge with a warning icon
3. WHEN the Agent hovers over the Compliance Flag Badge, THE System SHALL display a tooltip explaining the violated rule
4. THE System SHALL suggest an IRDAI-safe rewrite for flagged content
5. THE System SHALL block the Agent from sending messages that contain unresolved compliance violations

### Requirement 9

**User Story:** As an insurance agent, I want a map view with lead clusters, so that I can plan efficient field visits

#### Acceptance Criteria

1. WHEN the Agent opens the Maps page, THE System SHALL display a Google Maps embed with lead locations
2. THE System SHALL display lead clusters as light violet markers with lead count inside
3. THE System SHALL provide filters for Near Me, 30-minute radius, Follow-up pending, and High value leads
4. WHEN the Agent selects multiple leads, THE System SHALL display a Route Optimizer card with the optimized travel path
5. WHEN the Agent taps Start Route on the Route Optimizer card, THE System SHALL launch turn-by-turn navigation

### Requirement 10

**User Story:** As an insurance agent, I want AutoPilot mode to automate routine tasks, so that I can focus on customer conversations

#### Acceptance Criteria

1. WHEN the Agent enables AutoPilot, THE System SHALL continuously cycle through a prioritized queue of Next Best Actions
2. WHEN AutoPilot processes a lead, THE System SHALL open the lead profile automatically and present a suggested script
3. WHEN AutoPilot proposes a CRM update, THE System SHALL display the change in the Activity Feed with Apply, Edit, Skip, and Undo buttons
4. WHERE Auto-apply safe updates is enabled, THE System SHALL automatically apply non-outbound CRM updates such as tagging and reminders
5. WHEN AutoPilot proposes an outbound message, THE System SHALL require explicit Agent approval before sending

### Requirement 11

**User Story:** As an insurance agent, I want full visibility into AutoPilot actions, so that I maintain control and trust the automation

#### Acceptance Criteria

1. WHILE AutoPilot is running, THE System SHALL display a persistent mini-panel showing current client name, progress indicator, and next action
2. THE System SHALL display all queued AutoPilot actions in the Activity Feed panel with action summary cards
3. WHEN the Agent taps Pause on the AutoPilot mini-panel, THE System SHALL halt all automated actions within 1 second
4. THE System SHALL provide granular toggles for Auto-apply CRM updates, Auto-send templated messages, and Auto-open profiles
5. THE System SHALL log all AutoPilot actions with timestamp, rationale, confidence score, and Agent decision to an audit trail

### Requirement 12

**User Story:** As an insurance agent, I want AutoPilot to pause on low-confidence or risky actions, so that I can review critical decisions

#### Acceptance Criteria

1. WHEN AutoPilot generates an action with confidence below 70 percent, THE System SHALL require Agent confirmation before proceeding
2. WHEN the Compliance Guard flags an AutoPilot action, THE System SHALL pause AutoPilot and surface a mandatory rewrite flow
3. WHEN AutoPilot suggests a policy-adjacent update, THE System SHALL require human approval before execution
4. THE System SHALL display the confidence score and compliance status for each queued action in the Activity Feed
5. WHEN the Agent skips an AutoPilot action, THE System SHALL log the skip decision and proceed to the next queued action

### Requirement 13

**User Story:** As an insurance agent, I want AI-powered call and chat summaries, so that I can quickly review past interactions

#### Acceptance Criteria

1. WHEN the Agent requests a summary, THE System SHALL generate a summary using the AI Engine in the standard format with Summary, Insights, and Next Action sections
2. THE System SHALL support summarization of individual calls, chat threads, and last 7 or 30 day periods
3. THE System SHALL extract entities such as lead name, age, location, product interest, and premium amounts from conversations
4. THE System SHALL update the Last Interaction Summary field in the CRM after each summarization
5. THE System SHALL complete summarization and display results within 3 seconds

### Requirement 14

**User Story:** As an insurance agent, I want document parsing capabilities, so that I can quickly digitize visiting cards and customer documents

#### Acceptance Criteria

1. WHEN the Agent uploads a visiting card image, THE System SHALL extract name, phone number, and email using OCR within 5 seconds
2. WHEN the Agent scans a masked Aadhaar QR code, THE System SHALL extract only the permitted demographic data while preserving privacy
3. WHEN document parsing completes, THE System SHALL create or update the lead record with extracted information
4. THE System SHALL display the extracted fields with confidence scores for Agent verification
5. THE System SHALL allow the Agent to edit extracted values before saving to the CRM

### Requirement 15

**User Story:** As an insurance agent, I want coaching and performance insights, so that I can improve my sales skills

#### Acceptance Criteria

1. WHEN the Agent completes a call, THE System SHALL provide an agent score based on tone, engagement, and compliance
2. THE System SHALL identify missed cues or opportunities in the conversation
3. THE System SHALL provide coaching tips with specific examples from the call
4. THE System SHALL display tone analysis showing empathy, confidence, and professionalism metrics
5. THE System SHALL store coaching insights in the Agent's profile for trend analysis

### Requirement 16

**User Story:** As an insurance agent, I want explainability for all AI suggestions, so that I understand and trust the recommendations

#### Acceptance Criteria

1. WHEN the AI Engine makes a recommendation, THE System SHALL display a short explanation of the reasoning
2. THE System SHALL highlight the source signals used for each recommendation such as lead behavior or interaction history
3. WHERE X-Ray mode is enabled, THE System SHALL display detailed confidence scores and decision factors
4. THE System SHALL cite specific CRM data points used in generating summaries or actions
5. THE System SHALL never display hallucinated or unverified information as fact

### Requirement 17

**User Story:** As an insurance agent, I want a notification center, so that I can stay on top of renewals, follow-ups, and compliance alerts

#### Acceptance Criteria

1. WHEN the Agent opens the Notifications page, THE System SHALL display category tabs for Renewal, Follow-up, Compliance, and Missed Calls
2. THE System SHALL display notification cards with category tag, summary, and quick action buttons
3. WHEN the Agent taps a quick action button, THE System SHALL execute the action and mark the notification as done
4. THE System SHALL display a notification count badge on the navigation rail icon
5. THE System SHALL send push notifications for high-priority items such as compliance alerts and hot lead activity

### Requirement 18

**User Story:** As a compliance officer, I want an audit trail of all AI actions, so that I can review agent activity for regulatory compliance

#### Acceptance Criteria

1. THE System SHALL persist all AI Engine decisions, prompts, and final actions with timestamps to an audit log
2. THE System SHALL record the Agent's decision for each AI suggestion as applied, edited, or skipped
3. THE System SHALL provide an admin audit UI for compliance reviews with filtering by date, agent, and action type
4. THE System SHALL display Compliance Guard flags and resolutions in the audit trail
5. THE System SHALL retain audit logs for a minimum of 3 years per IRDAI requirements

### Requirement 19

**User Story:** As an insurance agent, I want a visually modern and minimal interface, so that I can work efficiently without distraction

#### Acceptance Criteria

1. THE System SHALL use the Neo-Minimal AI Workspace design language with Violet Indigo primary color #6A5AE0
2. THE System SHALL apply 18 to 22 pixel border radius to all card components
3. THE System SHALL use Inter or SF Pro typography with 600 weight for headings
4. THE System SHALL apply soft VisionOS-style shadows with 0px 8px 20px rgba(0,0,0,0.08) to elevated components
5. THE System SHALL use thin-lined Feather-style icons throughout the interface

### Requirement 20

**User Story:** As an insurance agent, I want smooth micro-animations, so that the interface feels responsive and polished

#### Acceptance Criteria

1. WHEN the Agent hovers over a card, THE System SHALL apply a 4 pixel lift animation with 200 millisecond duration
2. WHEN the Agent focuses the command bar, THE System SHALL apply a soft glow animation
3. WHEN the Agent navigates between pages, THE System SHALL apply a fade and upward spring transition with 260 millisecond duration
4. WHEN the Agent scrolls through horizontal Work Zones, THE System SHALL apply a snap scroll animation
5. WHEN the voice assistant is listening, THE System SHALL display a pulsing glow animation on the floating bubble
